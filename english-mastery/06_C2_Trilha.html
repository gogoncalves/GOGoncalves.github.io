Unit 01 / 08 · Registers

Registers — switching tones.

A mesma ideia, 4 formas. Em board meeting você fala academic. Em standup você fala casual. Em Slack DM você fala slang. C2 = trocar registers sem fricção, escolhendo o tom certo pro contexto.

IdeaRegisterPhrasing tech
"O deploy quebrou prod"AcademicThe deployment introduced a regression in the production environment.
BusinessThe deploy caused an incident in production.
CasualThe deploy broke prod.
SlangDeploy nuked prod. Lol.

Unit 02 / 08 · Figures of speech

Figures of speech — tech metaphors.

"Technical debt", "garbage in, garbage out", "moving fast and breaking things", "the iceberg". Tech English é cheia de metáforas — entender e usar te coloca no nível de quem escreve em tech blogs e dá talks.

FiguraTech example
Metaphor"Technical debt" (debt = financial → applied to code)
Simile"Microservices are like Lego — flexible but fragile"
Hyperbole"This codebase is a thousand-year-old fossil"
Understatement"It's a slight performance issue" (when prod is down)
Irony"Great, another deploy on Friday"
Personification"The CI is angry today"
Metonymy"Silicon Valley said no" (the industry, not the place)
Litotes"It's not bad" = it's actually good

Unit 03 / 08 · Pragmatic

Pragmatic competence — implicature.

"Interesting choice" não significa "interessante" — significa "discordo educadamente". "I'll think about it" significa "não". Em tech leadership, ler implicature é essencial.

SurfaceReal meaning
"That's an interesting take"I disagree but I'm being polite.
"I'll think about it"It's a soft no.
"With all due respect"I'm about to disagree strongly.
"As you may know..."You should already know this.
"That's a brave decision"I think it's a bad idea.
"Per my last email..."I already told you this.
"Going forward..."Don't do that again.
"Let's circle back"I don't want to talk about this now (or ever).
"Friendly reminder"You're late, do it now.
"As discussed"Even if we never discussed it, I'm framing it as agreed.

Unit 05 / 08 · Complex syntax

Complex syntax — long-form writing.

RFCs, op-eds, design docs longos exigem sintaxe complexa: relative + participle, embedded clauses, parallelism, anticipatory "it". Sem isso, sua escrita fica truncada.

RecursoExemplo
Participle clausesWalking through the codebase, I noticed several anti-patterns.
Reduced relativesThe bug found in production was unrelated to the deploy.
Anticipatory "it"It is widely understood that microservices add complexity.
ParallelismBuild, measure, learn.
NominalizationThe decision to migrate was driven by scale concerns.
Embedded clausesThe proposal, which was drafted by SRE and approved by the EMs, will ship next quarter.

Unit 07 / 08 · Academic

Academic English — papers & RFCs formais.

Em RFCs sêniors, white-papers, design docs de altíssimo padrão você usa academic English: voz passiva, hedged claims, formal connectors, sem contractions, sem phrasal verbs casuais.

Casual → AcademicExemplo
look into → investigateWe will investigate the cause.
find out → ascertainWe must ascertain the impact.
show → demonstrate / illustrateThe data demonstrates a clear pattern.
think → posit / hypothesizeWe posit that latency is the bottleneck.
can't / don't (contractions)cannot / do not
I think → It is argued thatIt is argued that this approach is optimal.

Unit 08 / 08 · Speaking authority

Keynote & op-ed — speaking authority.

Em conferences, podcasts, technical articles, você combina: rhetorical questions, parallelism, calls to action, cultural references, hedged authority. É uma camada acima — onde tech leaders são identificados.

RecursoExemplo (keynote)
Rhetorical question"What if we built it differently?"
Triplets / Parallelism"Build, measure, learn."
Call to action"Let's redefine what's possible."
Anaphora (repetition)"We dare. We ship. We learn."
Concession + pivot"Yes, it's hard. But hard is not impossible."
Personal anecdote"5 years ago, I was a junior. Today, I'm here."
