Unit 01 / 08 · Sequence

Past Perfect — sequence in incidents.

"By the time the alarm went off, we had already deployed". Duas ações no passado, uma anterior à outra. Em postmortems você narra timeline o tempo todo — Past Perfect é a ferramenta.

FormaEstruturaExemplo tech
Afirmativahad + past participleWe had deployed before the alarm.
Negativahadn't + p.p.The team hadn't tested the edge case.
PerguntaHad + sujeito + p.p.Had you reviewed the PR?
Continuoushad been + V-ingWe had been monitoring for hours.

⚠ Sequência: ação A antes de ação B (ambas passado)

Pattern típico
By the time the user noticed, we had already rolled back.
(1) rolled back (Past Perfect = mais antigo) → (2) noticed (Past Simple = mais recente)
Marker words
before · after · by the time · when · already · just · never (before)

Unit 02 / 08 · Hypothetical present

2nd Conditional — hypothetical decisions.

"If we had more engineers, we would refactor". Situação irreal/improvável no presente. Em arch reviews você usa o tempo todo: "If our scale were 10x, what would we change?".

Cláusula IFResultadoExemplo tech
If + past simplewould + V (base)If we had more engineers, we'd refactor.
If + WERE (irreal)would + VIf I were CTO, I'd change the stack.
If + past simplecould / might + VIf we used Postgres, we could scale better.

⚠ "If I were" — subjuntivo nativo

Em hypotheticals, use "were" (não "was") com I/he/she/it.

If I were the EM, I'd ship faster. ✓ (subjuntivo)
If I was the EM, I'd ship faster. (aceito mas menos formal)

Unit 03 / 08 · Postmortem regrets

3rd Conditional — postmortem regrets.

"If we had caught it earlier, we wouldn't have lost data". Hipótese sobre o passado — algo que NÃO aconteceu. Em postmortems é onipresente: "If we had monitored X, this wouldn't have happened".

Cláusula IFResultadoExemplo tech
If + past perfectwould have + p.p.If we had tested, we wouldn't have shipped the bug.
If + past perfectcould/might have + p.p.If we had monitored, we could have caught it earlier.

⚠ NUNCA "would have" antes da vírgula

❌
If we would have known, we would have rolled back.
✅
If we had known, we would have rolled back.

Unit 04 / 08 · Mixed

Mixed Conditionals — past affecting now.

"If we had migrated last year, we wouldn't be paying high cloud bills now". Mistura past condition + present result. Comum em retros estratégicas e decisões long-term.

TipoEstruturaExemplo tech
Past condition → Present resultIf + past perfect, would + VIf we had hired more, we would be done by now.
Present condition → Past resultIf + past simple, would have + p.p.If she were a better leader, the project wouldn't have failed.

Unit 05 / 08 · Regrets

Wish / If only — retro regrets.

"I wish we had better monitoring", "If only we'd caught it earlier". Em retrospectives você expressa desejos e arrependimentos — wish + tempo verbal específico.

EstruturaSignificadoExemplo tech
wish + past simpledesejo presente (não tem)I wish we had more engineers.
wish + past perfectarrependimento passadoI wish we had tested earlier.
wish + would + Vreclamação / mudança comportamentoI wish the CI would stop being flaky.
wish + could + Vhabilidade que não temI wish I could write Rust.

⚠ Wish ≠ Want

want = quero (real). wish = queria (irreal/lamento).

Unit 06 / 08 · Modal perfects

Must / should / might / could have done.

Em postmortems: "We should have caught it" (deveríamos), "It must have been the deploy" (deve ter sido), "It might have been a config issue" (talvez tenha sido), "We could have prevented it" (poderíamos).

ModalSignificadoExemplo tech
must have + p.p.certeza positiva sobre o passadoIt must have been the deploy.
can't have + p.p.certeza negativaIt can't have been the network — that's stable.
might/may have + p.p.possibilidadeIt might have been a race condition.
could have + p.p.possibilidade / capacidade não usadaWe could have added retries.
should have + p.p.conselho retroativo / arrependimentoWe should have caught it.
shouldn't have + p.p.arrependimento (do que foi feito)We shouldn't have shipped on Friday.

Unit 07 / 08 · Relative

Relative Clauses — describing systems.

"The service that handles auth", "The bug which broke prod", "The PM whose idea this was". Em docs técnicos você descreve componentes, pessoas, eventos o tempo todo.

PronomeRefere-se aExemplo tech
whopessoas (sujeito)The dev who shipped this.
whompessoas (objeto, formal)The PM whom we briefed.
whichcoisas / sistemasThe service which broke.
thatpessoas OU coisas (informal)The bug that crashed prod.
whoseposseThe team whose service is down.
wherelugaresThe repo where the auth lives.
whentempoThe day when the outage happened.

⚠ Defining vs Non-defining (vírgulas)

Defining (sem vírgulas): info essencial. Non-defining (com vírgulas): info extra. Em non-defining NÃO use "that".

The service that handles auth ✓ (defining)
Our auth service, which handles 1M req/s, is critical ✓ (non-defining)

Unit 08 / 08 · Causative

Causative — having work done.

"We have it deployed by SRE", "I got the PM to approve", "We're having the auth refactored". Você não fez — alguém fez por você. Vital pra delegar e descrever workflows multi-time.

EstruturaSignificadoExemplo tech
have something donecontratar/mandar fazer (neutro)We have the auth deployed by SRE.
get something doneidem (informal)I got the PR reviewed.
have someone do somethingfazer alguém fazer (mando)I had John fix the bug.
get someone to do somethingconvencer alguémI got the PM to approve.
